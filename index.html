    ANS

    1. Three main differences between web 1.0, web 2.0 and web 3.0:

    a. Web 1.0 targets on content creativity of producer, web 2.0 targets on content creativity of users and producers while web 3.0 targets on linked data sets.
    b. Web 1.0 is a “read-only” web with very little interaction where consumer can exchange the information together, but it is not possible to interact with the website while Web 2.0 is a “read-write” web. Web 2.0 allows managing and assembling large global crowd with common interests in social interaction. Web 3.0 is a “read-write-execute” web, individually oriented, highly portable and provides User developed smart applications in terms of the characteristic at front end.
    c. In Web 1.0 search engines retrieve macro contents. Search is very fast, but many times results are inaccurate. In Web 2.0 search engines retrieve tags with micro contents. The process of tagging is manual, tedious and covers a small portion of the WWW. In Web 3.0 search engines retrieve micro content texts which were tagged automatically.

    Source: https://www.researchgate.net/publication/262562142_Incremental_Journey_for_World_Wide_Web_Introduced_with_Web_10_to_Recent_Web_50_-_A_Survey_Paper

    2. Searching “side effects of Advil” in each version of the web:

    a. Web 1.0: Web 1.0 search engines are fast but unreliable so to get the best result, the user would need to open the website of the specific pharmaceutical company that makes Advil and look for the side effects in their website.
    b. Web 2.0: In Web 2.0 the user can use search engines like Google and search “side effects of Advil” and get pretty good results that are relevant to Advil. 
    c. Web 3.0: The use of ML and NLP improves the quality of the results and so the user can search “Advil” and get really relevant results. In addition, the user can even decsribe the medication instead of searching the name and still get relevant results.
